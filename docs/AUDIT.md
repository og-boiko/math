## 1. Знайдені баги та виправлення

### 1.1 (критичний) `checkAnswer` ламав списки чисел

**Симптом.** У задачах на подвійні нерівності
(`equationsDoubleInequality`) правильна відповідь — це список
цілих, наприклад `1, 2, 3`. `normalizeAnswer` замінювала **всі**
коми на крапки → `1.2.3`. Дитина пише `1, 2, 3` — нормалізується
в `1.2.3`. Канонічна відповідь так само → `1.2.3`. Збігалось випадково,
але _будь-який_ інший порядок або роздільник давав неправильно
(наприклад `3, 2, 1`, або `1; 2; 3`).

**Виправлення.** `src/lib/format.ts`:
- `normalizeAnswer` залишився скаляром (числа, слова, дроби).
- Додані `normalizeNumberList` та `normalizeNumberListUnordered` для списків.
- `checkAnswer` тепер сам визначає тип за **канонічною** відповіддю
  (а не за введенням): якщо `correct` виглядає як список цілих
  через кому/крапку з комою → порівнюємо як невпорядковану множину.
- Для списків приймаємо роздільники: `,`, `;`, пробіл; будь-який порядок
  (`3, 1, 2` = `1, 2, 3`).
- `3,14` НЕ список — це десятковий, бо токен містить кому посередині числа.

**Що дає дитині.** На задачі «знайди всі цілі x: 1 < x ≤ 4»:
- `2, 3, 4` ✓
- `4, 3, 2` ✓
- `2;3;4` ✓
- `2 3 4` ✓
- `4` (одне число) ✗

### 1.2 Розходження `subtopic` у `oralMultiplication`

**Симптом.** Генератор оголошений як `subtopic: 'Таблиця множення'`,
а в задачі `task.subtopic = 'Множення'`. Це призводило до того, що
у журналі помилок і статистиці один і той самий тип задач рахувався
під двома різними іменами.

**Виправлення.** `src/features/tasks/generators/oral.ts`: уніфіковано на
`'Таблиця множення'` (відповідає тому, що дитина бачить у меню тем).

### 1.3 Потенційні нескінченні рекурсії в генераторах

Кілька генераторів використовували `return self.generate(...)` як спосіб
«перегенерувати» при невдалому випадковому збігу. Хоча на практиці
це працювало, це **антипатерн**:
- Збільшує stack-frame під час навантаження.
- Може робити downgrade рівня (як було в `logicAge` → fallback на рівень 2).
- Робить виконання непередбачуваним.

Усі замінено на детерміноване конструювання або на bounded loop
(до 8 спроб) з гарантованим fallback. Виправлено в:

| Файл | Проблема | Як виправлено |
|---|---|---|
| `equations.ts` → `equationsBrackets` | `(x − a) ÷ c = b`: рекурсія, поки `(xV − a)` не стане кратним `c` | Тепер `xV = b · c + a` — детерміновано. |
| `equations.ts` → `equationsDoubleInequality` | Рекурсія на порожньому розв'язку | Bounded loop + fallback `1 ≤ x ≤ 3`. |
| `logic.ts` → `logicAge` (рівень 4-5) | Лінійний пошук + downgrade на рівень 2 | Аналітично: `dad = 2·son + (k−1)·yearsAhead`. |
| `geometry.ts` → `geometryAngles` (рівень 4-5) | Рекурсія, поки результат не в коридорі | Спочатку `c`, потім `a` у безпечному діапазоні. Для рівня 5 — `apex` парний. |
| `geometry.ts` → `geometryRectangleSameArea` | Рекурсія, якщо випадково обрано просте число | Bounded loop + fallback `12 = 1×12 = 2×6 = 3×4`. |
| `word.ts` → `wordJointWork` | Рекурсія, якщо дроби не дають ціле | Bounded loop по `pool` + fallback. |
| `order.ts` → `orderNoBrackets` | Рекурсія з `difficulty=1` як fallback | Bounded loop + fallback `a + b`. |
| `order.ts` → `orderWithBrackets` | Рекурсія з `difficulty=1` як fallback | Bounded loop + fallback `(a + b) × c`. |
| `column.ts` → `columnCombined` | Рекурсія | Bounded loop + fallback. |

### 1.4 `decimalRead` ламався на рівнях 4-5

**Симптом.** Допоміжна функція `intWord` обробляла лише числа 0-99,
а на рівнях 4-5 `fracPart` (тисячна частина) міг бути 100-999.
Як наслідок — у питанні з'являлося `undefined тисячних`.

**Виправлення.** Розширено `intWord` до 0-999 (додано `hundreds[]`
з `сто, двісті, ..., дев'ятсот` і композитну логіку).

## 3. Конкурентний ландшафт (4 клас, математика, ігровий тренажер)

### Khan Academy (Khanmigo, Mastery Maps)

- **Безкоштовно**, відомий бренд, охоплює всю програму.
- **Mastery learning**: тема відкривається, коли попередню підтверджено.
- **Адаптивна складність**: помилка → задача того ж типу легша, успіх → наступний рівень.
- **Khanmigo (AI)**: GPT-туторій, який може вести крок за кроком, питати «чому?», показувати контр-приклад.
- **Прогрес**: дерево навичок (skill tree), процентна шкала по темах, бейджі.
- **Слабке місце для українського ринку**: немає укр. локалізації для початкової школи, вікторини «академічні», без сюжету.

### Prodigy

- **Сильна гейміфікація**: дитина — герой у RPG-фентезі, бої = задачі.
- **Адаптивна складність** на льоту, автоматичне калібрування.
- **Параметри для батьків / вчителів**: pacing, цілі, звіти.
- **Quests / events**: тимчасові події, нові локації, нові предмети — двигун повернення.
- **Слабке місце**: монетизація (Premium) розблоковує більшу частину гри, дитина без Premium швидко впирається.

### SplashLearn

- **Візуальні мікроігри**: кожна задача — окрема анімована мінігра (drag-n-drop, розфарбовка, віртуальні монети).
- **Адаптивна складність + spaced repetition**.
- **Прогрес-карта по класах**: видно, що далі.
- **Сильна аналітика для батьків**: тижневий звіт, на чому застрягає.
- **Слабке місце**: задачі здебільшого на drill, мало складених задач і логіки.

### Що ми робимо краще

- **Українська мова, ДУМ, формат ЗНО/ДПА** — їх немає в жодного.
- **Робочий стіл**: чернетка, калькулятор-стрічка, стовпчик — це наш unique selling point. У жодного конкурента немає інструменту, що повторює фізичний зошит.
- **Складені задачі, нерівності, ребуси** — у нас є, у конкурентів частково.
- **Темна сторона**: немає сюжету / світу / прогресування персонажа, як у Prodigy.

## 4. Подальші кроки (рекомендації)

Згруповано за вартістю і ефектом. **Ⓡ** = ризик, **Ⓔ** = ефект для дитини.

### 4.1 Швидкі виграші (1-2 дні)

1. **Адаптивна складність всередині сесії.** Зараз `currentDifficulty`
   міняється раз на сесію. Можна після кожних 3-х відповідей робити
   корекцію: 3 правильні → +1 рівень, 2 з 3 неправильні → −1 рівень.
   _Ⓔ високий, Ⓡ низький._ Реалізація — у `awardForAnswer` в `profile.ts`.

2. **Відкриті відповіді з частковим зарахуванням.** Зараз для
   подвійних нерівностей: «1, 2, 4» при правильному «1, 2, 3» →
   `wrong`. Краще було б показати: «у тебе 2 правильні з 3 — спробуй
   ще». Те саме для багатокрокових текстових. Зробити поверх нової
   `normalizeNumberListUnordered`.
   _Ⓔ середній, Ⓡ низький._

3. **Час як параметр зірочок.** Зараз 3⭐ = з першого разу без
   підказок. Можна додати: 3⭐ + бонусна зірка / монета, якщо вкладеш
   у `estimatedSec`. Це підкине інтерес у тих, хто вже все знає.
   _Ⓔ середній, Ⓡ низький._

4. **Індикатор «гарячої смуги» (streak).** На картці задачі —
   маленький лічильник «🔥 3 поспіль». Втрата при помилці.
   _Ⓔ середній, Ⓡ нульовий._

### 4.2 Середні (тиждень)

5. **Daily quest / weekly quest.** Як у Prodigy: «реши 5 задач на
   рух за тиждень → +50 монет, новий тематичний фон». Реалізація —
   нова сутність `quests` у профілі, перевірка в `awardForAnswer`.
   _Ⓔ високий, Ⓡ середній (треба контент)._

6. **Heatmap-календар активності.** Сторінка `/calendar` уже існує,
   але хочеться GitHub-style heatmap по днях, з підсвічуванням
   streak-ів. Це потужний візуальний хук на повернення.
   _Ⓔ середній, Ⓡ низький._

7. **Spaced repetition для помилок з кривою забування.** Зараз `nextReviewDate`
   рухається лінійно. Краще — SM-2 (як в Anki): успішне повторення
   подвоює інтервал, помилка скидає до 1 дня. Поліпшить довгостроковий
   ефект.
   _Ⓔ високий, Ⓡ низький._

8. **Подкази-ілюстрації для геометрії.** Поки що задачі на периметр /
   площу — текстові. Якщо додати SVG-візуалізацію прямокутника /
   трикутника / Г-фігури, дитині буде легше «бачити».
   _Ⓔ високий, Ⓡ середній (потрібно час на SVG)._

### 4.3 Великі (1-2 тижні)

9. **Сюжетна оболонка** (як Prodigy, але простіше). Дитина — астронавт,
   який «літає» між планетами (теми). Кожна задача = крок до нової
   планети. Прогрес-кільце замість лінійки. Можна зробити поверх
   існуючих `themes` без переробки.
   _Ⓔ дуже високий (retention), Ⓡ високий (треба арт)._

10. **AI-туторій (на кшталт Khanmigo) у вигляді чату.** Якщо дитина
    помилилась двічі — кнопка «🤖 Поясни мені». Опціонально, тільки
    при наявності інтернету. На бекенді — Edge Function в Supabase
    з виклик OpenAI/Claude/Gemini. Промпт: «Поясни 4-класнику крок за
    кроком: …».
    _Ⓔ високий, Ⓡ високий (вартість API, етика, GDPR)._

11. **Battle-режим (PvE/PvP).** Дві дитини або дитина-проти-бота
    розв'язують той самий набір — хто швидше і точніше. Можна почати
    з **async PvP** (грає окремо, порівнюються час і точність), що
    знімає WebSocket-залежність.
    _Ⓔ високий, Ⓡ високий (треба бекенд лоббі)._

12. **Звіт для батьків (PDF або dashboard).** Є lots of даних у
    `profile.ts` (помилки, теми, час). Раз на тиждень — згенерувати
    «звіт прогресу» з діаграмами. Це потужний канал для рекомендацій
    знайомим.
    _Ⓔ середній, Ⓡ середній._
