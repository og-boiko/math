# MathQuest — Технічне завдання (v2.2)

## 1. Огляд

**MathQuest** — веб-застосунок (PWA) для тренування математики. Цільова
аудиторія: діти 8–12 років, мова інтерфейсу — українська. Гейміфікований
підхід: щоденні сесії 15–20 хв, теми зі шкільної програми 4 класу +
місток у 5 клас.

З **v2.0** (червень 2026) додано опціональну **хмарну синхронізацію через
Supabase**: один email-акаунт = один профіль, доступний з кількох пристроїв.

З **v2.2** (червень 2026) додано **публічна SEO-оптимізована лендінг-сторінка** (`/`)
для залучення користувачів. Додаток переміщено на `/app/*` з кодом-сплітингом:
лендінг завантажується лініво (33 KB / 9 KB gzip), додаток тільки за потреби.

## 2. Цілі та не-цілі

### Цілі

- Щоденна практика 15–20 хв без примусу
- Покрити усі ключові теми 4 класу + місток у 5 клас
- Автоматично повертати помилки для опрацювання
- Cross-device sync без болю налаштувань
- **Привабити нових користувачів через публічну лендінг-сторінку**
- **Оптимізуватися для SEO: index, follow, social sharing**
- **Розділити маркетинг (/) від додатку (/app) для UX & performance**
- Працювати **повністю офлайн** коли немає інтернету або акаунту

### Не-цілі (поки)

- Мультиплеер у реальному часі
- Платні підписки / магазин
- Озвучка / TTS
- Адмінська батьківська панель (видалена в v2 — UX-спрощення)

## 3. Користувачі

### Дитина (8–12 років) — основна

- Любить тематичні сетинги: космос, динозаври, країни, voxel, Roblox-style
- Користується переважно з телефона
- Реєструється сама через email + пароль АБО грає в "гостьовому" режимі

### НЕ-користувачі

- Окремих "батьківських" акаунтів немає. Дитина = власник профілю.
- Якщо потрібно показати прогрес родичам — публічний лідерборд (opt-in).

## 4. Тематичні сетинги

При першому запуску дитина обирає сетинг, який змінює візуал/назви/локації.

| ID | Назва | Аватар | Локації | Бос | Валюта |
|---|---|---|---|---|---|
| `space` | Космос | Астронавт | Планети | Чорна діра | Кристали |
| `dino` | Динозаври | Палеонтолог | Епохи | Тиранозавр | Бурштин |
| `world` | Подорож світом | Мандрівник | Континенти | Бермудський трикутник | Монети |
| `voxel` | Воксельний світ | Будівельник | Біоми | Ендер-дракон | Смарагди |
| `blocky` | Блочний світ | Аватар-блок | Обби-світи | Бос-обба | Зірочки |

Не використовуємо офіційні текстури/логотипи — тільки "inspired by".

## 5. Функціональні вимоги

### 5.1. Онбординг

- REQ-001: Welcome екран з 2 опціями (cloud / local).
- REQ-002: Cloud — `/auth` з signup-формою (email + password + ім'я + вік + сетинг).
- REQ-003: Local — миттєвий профіль у localStorage.
- REQ-004: Запит установки PWA через manifest + custom prompt (опціонально).

### 5.2. Головний екран (Hub)

- Картка «Вільне тренування»
- Картка «Робота над помилками» (якщо є черга)
- Картка «Контрольні»
- Кнопки: Карта світу, Лідерборд (якщо cloud), Профіль, Налаштування
- Бейдж стріку

### 5.3. Вільне тренування

10 тем, кожна з 1–5 рівнями. Сесія = 10 задач.

### 5.4. Екран задачі

- Великий текст, числова клавіатура для `number`-задач, варіанти для choice
- «Перевірити» → правильно / помилка / розкриття
- Прогрес-бар сесії

### 5.5. Підказки

- 2 рівні + повне рішення
- 1-ша спроба без підказки = 3⭐
- 1 підказка = 2⭐, 2 підказки = 1⭐
- Розкриття = 0⭐

### 5.6. Робота над помилками

Інтервальне повторення: 1, 2, 4, 8, 16 днів. 5 успіхів поспіль → задача
видаляється з черги.

### 5.7. Гейміфікація

- **Зірки:** 3/2/1/0 за задачу
- **XP:** 10 за зірку, рівень `√(xp/100)+1`
- **Монети:** 5 за сесію + 1 за кожну правильну
- **Streak:** +1 за день з виконаною ціллю
- **Ачивки:** ≥15 умов з автоматичною перевіркою

### 5.8. Адаптивна складність

3 правильні поспіль → +1 рівень. 2 помилки в межах 5 → −1.

### 5.9. Карта світу

Тематична візуалізація прогресу, відкриття локацій по дереву залежностей.

### 5.10. Календар

Heat-map за останні 12 тижнів, дотик → деталі дня.

### 5.11. Профіль

Аватар, рівень, точність по темах, ачивки.

### 5.12. Налаштування

Звуки, денна ціль, сетинг, акаунт у хмарі (logout), opt-in лідерборду,
скидання профілю.

### 5.13. Лідерборд (cloud-only)

3 таби: 🆙 Приріст тижня · 🔥 Streak · 🌟 Сезон. Показує лише opt-in профілі.
Власний ранг — окрема картка зверху.

### 5.14. Звуки

4 ефекти (правильно / помилка / завершення / новий рівень) через Web Audio
API без зовнішніх файлів.

### 5.15. Auth (v2)

- Email + password
- Confirm-email вимкнено в Supabase (instant signup)
- Min password 6 символів
- Logout очищає Supabase-сесію + локальний zustand

### 5.16. Робочий стіл (v2.1.1)

Bottom-sheet, що відкривається кнопкою на екрані задачі. Три інструменти:

- **🧮 Обчислення (CalcStrip):** багаторядковий калькулятор. `+ − × ÷ ( )`,
  кома як десятковий, кнопка `↑ ans` для повторного використання попереднього
  результату. Парсер — на `lib/math-eval.ts`, без `eval`.
- **📐 Стовпчик (ColumnGrid):** структурована сітка розрядів для
  додавання/віднімання/множення/ділення. Витягує операнди з `task.question`.
- **✏️ Чернетка (Scratchpad):** Canvas 2D з PointerEvents і pressure-sensitivity.
  4 кольори, гумка.

Дефолтний таб обирається за `topicId` (column → 📐, geometry/logic → ✏️,
решта → 🧮). Стан зберігається в memory store за `taskId` — між табами і
відкриттями за час задачі. Очищається при переході на наступну задачу.

Лічильник використань (`flags.workbenchUses`) інкрементується 1 раз на задачу
і дає 2 ачивки. Toggle вмикання — в `/settings` (дефолт `true`).

## 7. Технічна архітектура

### Стек

**Frontend:**
- Vite 5 · React 18 · TypeScript 5
- Tailwind CSS 3 · tailwind-merge
- Zustand 4 (persist + migration)
- React Router 6
- lucide-react · vite-plugin-pwa
- @supabase/supabase-js v2 (no-op без env)

**Backend (опціонально):**
- Supabase Postgres + Auth + RLS
- 7 SQL-міграцій (без edge functions у v2)

### Шари

```
UI (React) → Store (Zustand) → Generators / Cloud API → Supabase RPC
                ↑                        ↓
          persist localStorage    debounced cloudSync
```

### Модель даних — frontend

- `src/store/profile-types.ts` — типи (винесено окремо для уникнення циклу)
- `src/store/profile.ts` — Zustand store + persist
- `src/store/session.ts` — активна сесія

### Модель даних — Supabase

- `families` + `family_members` — деталь реалізації, фронт її не бачить
- `child_profiles` — 1 рядок на акаунт
- `profile_stats` — денормалізовані агрегати
- `topic_progress` — 10 рядків на профіль
- `daily_activity` — 1 рядок на день
- `error_queue` — журнал помилок
- `leaderboard_*` — 3 матеріалізовані таблиці

### RPC API (security definer)

- `get_or_create_my_profile(name, age, theme)` → bootstrap
- `get_my_full_snapshot()` → full hydrate
- `upsert_my_stats(...)` · `upsert_my_topics(jsonb)` · `upsert_my_activity(...)` · `replace_my_errors(jsonb)` → push
- `set_my_leaderboard_opt_in(boolean)` → toggle
