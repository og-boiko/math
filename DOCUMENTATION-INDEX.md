## 🔄 Documentation Workflow

### When Adding a New Feature

1. **Design phase**: Update `docs/SPEC.md`
2. **Implementation**: Update `docs/ROADMAP.md` (mark item done)
3. **Content**:
   - If affects landing → update `docs/LANDING.md`
   - If affects tasks → update `docs/CONTENT.md`
   - If affects dev setup → update `DEVELOPER-GUIDE.md`
4. **Testing**: Update `LAUNCH-CHECKLIST.md` if needed
5. **Review**: Ensure `README.md` still accurate

### When Deploying to Production

1. Complete **`LAUNCH-CHECKLIST.md`** (50+ items)
2. Verify all tests pass (`npm test`)
3. Check build (`npm run build`)
4. Review SEO (Open Graph, robots.txt, sitemap)
5. Deploy & verify staging
6. Deploy to production
7. Monitor metrics (Analytics, error logs)

### When Onboarding New Developers

1. Give them **`DEVELOPER-GUIDE.md`** (start here)
2. Have them read **`README.md`** (overview)
3. Have them run `npm run dev` (hands-on)
4. Point them to **`docs/SPEC.md`** (deep dive)
5. Have them review **`docs/LANDING.md`** (for landing work)

### Most Important Commands

```bash
# Development
npm install        # Install dependencies
npm run dev        # Start dev server (http://localhost:5174/)

# Testing
npm test           # Run all tests (71 items)
npm run test:workbench    # Test UI components
```
